Codigo destaque

import time
from dataclasses import dataclass

@dataclass
class BenchResult:
    model: str
    latency_ms: float
    tokens_per_sec: float
    cost_usd: float
    quality_score: float

def benchmark_model(provider, model: str, prompts: list[str]) -> BenchResult:
    """Roda suite de prompts e coleta metricas."""
    latencies, tps_list = [], []

    for prompt in prompts:
        t0 = time.perf_counter()
        resp = provider.generate(model, prompt)
        elapsed = (time.perf_counter() - t0) * 1000

        latencies.append(elapsed)
        tokens = count_tokens(resp.text)
        tps_list.append(tokens / (elapsed / 1000))

    quality = llm_judge_score(prompts, resp.text)
    cost = provider.estimate_cost(model, prompts)

    return BenchResult(
        model=model,
        latency_ms=sum(latencies) / len(latencies),
        tokens_per_sec=sum(tps_list) / len(tps_list),
        cost_usd=cost,
        quality_score=quality,
    )

O que aprendi

  • LLM-as-judge — usar um modelo forte para avaliar outputs de modelos menores e surpreendentemente consistente.
  • Ollama para local — modelos 7B locais tem latencia competitiva e custo zero para tarefas simples.
  • Tokens/sec vs latencia — TTFT (time to first token) importa mais que throughput para UX.
  • Dashboards Streamlit — visualizacao interativa torna benchmarks acionaveis para decisoes de negocio.
