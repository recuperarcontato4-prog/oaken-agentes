CLI Multi-Provider LLM Helper

CLI que roteia tarefas para OpenAI, Gemini ou Anthropic com auto-detect de provider e fallback inteligente.

Codigo destaque

from enum import Enum
import typer

class Provider(Enum):
    OPENAI = "openai"
    GEMINI = "gemini"
    ANTHROPIC = "anthropic"

def route_request(prompt: str, provider: Provider = None) -> str:
    """Roteia prompt para o provider selecionado ou auto-detecta."""
    if provider is None:
        provider = auto_detect(prompt)

    clients = {
        Provider.OPENAI: call_openai,
        Provider.GEMINI: call_gemini,
        Provider.ANTHROPIC: call_anthropic,
    }
    handler = clients[provider]
    try:
        return handler(prompt)
    except Exception:
        # fallback para proximo provider disponivel
        for alt in Provider:
            if alt != provider:
                return clients[alt](prompt)

O que aprendi

  • Multi-provider abstraction — isolar cada SDK atras de interface comum elimina lock-in e simplifica testes.
  • Auto-detect heuristics — analisar tokens, custo e contexto do prompt para escolher o melhor modelo automaticamente.
  • CLI ergonomics com Typer — type hints do Python viram flags/argumentos com zero boilerplate.
  • Fallback resiliente — try/except em cadeia garante resposta mesmo quando um provider falha.
