Moderação IA

Sistema multi-agente de moderação de conteúdo com Human-in-the-Loop — classificador, revisor e decisor com interrupt para casos ambíguos.

LangGraph HITL Multi-Agent SQLite

Arquitetura

┌────────────┐     ┌──────────────┐     ┌────────────┐
│  Content   │────▶│ Classifier   │────▶│  Score     │
│  Input     │     │ Agent        │     │  0.0-1.0   │
└────────────┘     └──────────────┘     └─────┬──────┘

Código destaque

from langgraph.graph import StateGraph, END
from langgraph.checkpoint.sqlite import SqliteSaver
import sqlite3

# Lazy singleton para checkpoint
_db: sqlite3.Connection | None = None

def get_checkpointer():
    global _db
    if _db is None:
        _db = sqlite3.connect("moderation.db", check_same_thread=False)
    return SqliteSaver(_db)

def classify(state) -> dict:
    """Classifica conteúdo com score de toxicidade."""
    score = llm.invoke(CLASSIFY_PROMPT.format(
        content=state["content"]
    ))
    return {"toxicity_score": float(score)}

def route_by_score(state) -> str:
    score = state["toxicity_score"]
    if score > 0.8: return "auto_block"
    if score < 0.3: return "auto_approve"
    return "human_review"  # zona cinzenta

graph = StateGraph(ModerationState)

O que aprendi

  • interrupt_before — Pausa o grafo antes de um nó, permitindo intervenção humana assíncrona.
  • update_state — Injectar decisão humana no state e retomar execução do grafo.
  • Conditional edges — Routing por score cria caminhos distintos para auto/manual sem if/else.
  • Lazy singleton — Conexão DB lazy evita problemas de threading e garante checkpoint persistente.
