Eval Prompts Framework

Framework de A/B test de prompts com metricas quantitativas: latencia, custo, qualidade e consistencia.

Python Pydantic YAML pytest

Codigo destaque

from pydantic import BaseModel
import yaml, time

class PromptVariant(BaseModel):
    name: str
    template: str
    model: str = "gpt-4o-mini"

class EvalConfig(BaseModel):
    variants: list[PromptVariant]
    test_cases: list[dict]
    iterations: int = 5

def run_eval(config: EvalConfig) -> dict:
    """Executa A/B test e retorna metricas por variante."""
    results = {}
    for variant in config.variants:
        metrics = {"latency": [], "cost": [], "scores": []}
        for case in config.test_cases:
            for _ in range(config.iterations):
                t0 = time.perf_counter()
                resp = call_llm(variant.template.format(**case))
                elapsed = time.perf_counter() - t0
                metrics["latency"].append(elapsed)
                metrics["scores"].append(score_response(resp, case))
        results[variant.name] = aggregate(metrics)
    return results

O que aprendi

  • Prompt engineering cientifico — medir antes de decidir; intuicao engana, metricas nao.
  • Pydantic para configs — validacao de schemas YAML com tipos fortes evita erros silenciosos.
  • Estatistica de LLMs — variancia alta exige multiplas iteracoes para resultados confiaveis.
  • pytest como harness — cada eval vira um test case, CI/CD barra regressoes de prompt.
