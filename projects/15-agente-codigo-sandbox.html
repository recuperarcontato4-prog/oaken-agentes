Agente Código Sandbox

Loop write → run → fix dentro de container Docker isolado. O agente escreve código, executa, analisa erros e corrige automaticamente.

Código destaque

import docker
from langgraph.graph import StateGraph, END

client = docker.from_env()

def run_in_sandbox(code: str, timeout: int = 30) -> dict:
    """Executa código num container efémero."""
    container = client.containers.run(
        image="python:3.11-slim",
        command=["python", "-c", code],
        mem_limit="128m",
        network_disabled=True,
        detach=True,
    )
    result = container.wait(timeout=timeout)
    logs = container.logs().decode()
    container.remove(force=True)
    return {
        "exit_code": result["StatusCode"],
        "output": logs
    }

def should_fix(state) -> str:
    """Decide se precisa corrigir ou finalizar."""
    if state["exit_code"] != 0 and state["attempts"] < 3:
        return "fix"
    return "done"

O que aprendi

  • Sandbox seguro — Containers sem rede e com limite de memória previnem execução maliciosa.
  • Code generation — LLMs escrevem código melhor quando recebem o traceback completo do erro.
  • Auto-fix loop — Limitar tentativas (max 3) evita loops infinitos em bugs impossíveis para o LLM.
  • Containers efémeros — Criar e destruir containers por execução garante isolamento total.
