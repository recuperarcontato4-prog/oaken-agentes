RAG Multimodal CLIP

Busca cross-modal: pesquise com texto e encontre imagens, ou vice-versa, usando embeddings CLIP.

CLIP ChromaDB PyTorch Transformers

Arquitetura

  Indexacao:
  +-----------+     +-----------+
  |  Images   |     |  Texts    |
  +-----+-----+     +-----+-----+
        |                  |
  +-----v------------------v-----+
  |       CLIP Encoder           |
  |  (shared embedding space)    |
  +-----+------------------------+
        |
  +-----v-----------+
  |   ChromaDB      |
  |   Vector Store  |
  +------------------+

  Busca:
  Query (text OR image)
        |
  +-----v-----------+
  |  CLIP Encode    |
  +-----+-----------+
        |
  +-----v-----------+
  |  Cosine Search  |
  |  top_k results  |
  +------------------+

Codigo destaque

import torch
from transformers import CLIPModel, CLIPProcessor
from PIL import Image
import chromadb

model = CLIPModel.from_pretrained("openai/clip-vit-base-patch32")
processor = CLIPProcessor.from_pretrained("openai/clip-vit-base-patch32")

def encode_image(img_path: str) -> list[float]:
    """Gera embedding de imagem via CLIP."""
    image = Image.open(img_path)
    inputs = processor(images=image, return_tensors="pt")
    with torch.no_grad():
        emb = model.get_image_features(**inputs)
    return emb.squeeze().tolist()

def encode_text(text: str) -> list[float]:
    """Gera embedding de texto via CLIP."""
    inputs = processor(text=[text], return_tensors="pt")
    with torch.no_grad():
        emb = model.get_text_features(**inputs)
    return emb.squeeze().tolist()

def search(query: str, collection, top_k: int = 5):
    """Busca cross-modal: texto busca imagens."""
    q_emb = encode_text(query)
    results = collection.query(query_embeddings=[q_emb], n_results=top_k)
    return results

O que aprendi

  • Espaco compartilhado de embeddings — CLIP projeta texto e imagem no mesmo espaco vetorial, permitindo busca cross-modal.
  • Normalizacao importa — normalizar embeddings antes do cosine similarity melhora drasticamente os resultados.
  • Trade-off modelo vs velocidade — ViT-B/32 e rapido; ViT-L/14 e mais preciso mas 4x mais lento.
  • Batch processing — indexar milhares de imagens exige batching + GPU para viabilizar.
