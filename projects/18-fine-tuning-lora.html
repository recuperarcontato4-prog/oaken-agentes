Fine-Tuning QLoRA

Fine-tuning eficiente com QLoRA 4-bit — adapta LLMs de 7B parâmetros numa GPU consumer com menos de 6 GB VRAM.

Transformers PEFT QLoRA

Código destaque

from transformers import AutoModelForCausalLM, BitsAndBytesConfig
from peft import LoraConfig, get_peft_model, prepare_model_for_kbit_training
from trl import SFTTrainer

# Quantização 4-bit NF4
bnb_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_quant_type="nf4",
    bnb_4bit_compute_dtype=torch.bfloat16,
    bnb_4bit_use_double_quant=True,
)

model = AutoModelForCausalLM.from_pretrained(
    "mistralai/Mistral-7B-v0.1",
    quantization_config=bnb_config,
    device_map="auto",
)
model = prepare_model_for_kbit_training(model)

# LoRA adapters — só 0.1% dos parâmetros
lora_config = LoraConfig(
    r=16, lora_alpha=32, lora_dropout=0.05,
    target_modules=["q_proj", "v_proj", "k_proj", "o_proj"],
    task_type="CAUSAL_LM",
)
model = get_peft_model(model, lora_config)
model.print_trainable_parameters()
# trainable params: 4,194,304 || all: 7,241,748,480 || 0.058%

O que aprendi

  • Quantização NF4 — Normal Float 4-bit preserva distribuição dos pesos melhor que INT4.
  • LoRA adapters — Treinar <0.1% dos parâmetros atinge 95%+ da qualidade de full fine-tune.
  • Treino eficiente — Gradient checkpointing troca compute por VRAM, cabendo em GPUs consumer.
  • Merge + push — Adapters podem ser merged no modelo base para deploy sem overhead de inferência.
