Agente ReAct Zero-Shot

Loop ReAct (Reason + Act) com 3 ferramentas: busca web, calculadora e Wikipedia. Sem fine-tuning.

ReAct DuckDuckGo numexpr LLM

Codigo destaque

import re
from duckduckgo_search import DDGS
import numexpr

TOOLS = {
    "search": lambda q: DDGS().text(q, max_results=3),
    "calc": lambda expr: str(numexpr.evaluate(expr)),
    "wiki": lambda q: fetch_wikipedia(q),
}

def react_loop(query: str, max_steps: int = 8) -> str:
    """Loop ReAct: Thought -> Action -> Observation."""
    history = f"Question: {query}\n"

    for step in range(max_steps):
        response = call_llm(REACT_PROMPT + history)

        # Extrair acao do response
        if "Final Answer:" in response:
            return response.split("Final Answer:")[1].strip()

        match = re.search(r"Action: (\w+)\[(.+?)\]", response)
        if match:
            tool, arg = match.groups()
            observation = TOOLS[tool](arg)
            history += f"{response}\nObservation: {observation}\n"

    return "Limite de passos atingido."

O que aprendi

  • ReAct pattern — intercalar raciocinio e acao permite ao LLM decompor problemas complexos passo a passo.
  • Tool dispatch por regex — parsear "Action: tool[arg]" e simples e robusto para agentes basicos.
  • Limite de steps — sem max_steps, o agente pode entrar em loop infinito; guardrail essencial.
  • Zero-shot vs few-shot — GPT-4o-mini funciona zero-shot com o prompt certo; modelos menores precisam de exemplos.
