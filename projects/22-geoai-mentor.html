GeoAI Mentor

Chatbot mentor especializado para geocientistas — usa LCEL chains com session memory e persona de geólogo sénior.

LangChain OpenAI LCEL Memory

Arquitetura

┌────────────┐     ┌──────────────┐     ┌────────────┐
│  User      │────▶│  Session     │────▶│  LCEL      │
│  Message   │     │  Memory      │     │  Chain     │
└────────────┘     │  (per user)  │     └─────┬──────┘
                   └──────────────┘           │
                                        ┌─────▼──────┐
                   ┌──────────────┐     │  System    │
                   │   OpenAI    │◀────│  Prompt    │
                   │   GPT-4o   │     │  (Geólogo) │
                   └──────┬───────┘     └────────────┘
                          │
                   ┌──────▼───────┐
                   │  Response   │
                   │  + Memory   │
                   │  Update     │
                   └──────────────┘

Código destaque

from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder
from langchain_core.runnables.history import RunnableWithMessageHistory
from langchain_community.chat_message_histories import ChatMessageHistory

# Store de sessões
sessions: dict[str, ChatMessageHistory] = {}

def get_session(session_id: str) -> ChatMessageHistory:
    if session_id not in sessions:
        sessions[session_id] = ChatMessageHistory()
    return sessions[session_id]

# Prompt com persona de geólogo sénior
prompt = ChatPromptTemplate.from_messages([
    ("system", """Você é um geólogo sénior com 20 anos de
experiência em exploração de petróleo. Responda de forma
didática, use analogias práticas e sempre relacione
conceitos teóricos com cenários de campo."""),
    MessagesPlaceholder(variable_name="history"),
    ("human", "{input}"),
])

# LCEL chain com memory
chain = prompt | ChatOpenAI(model="gpt-4o-mini", temperature=0.7)

mentor = RunnableWithMessageHistory(
    chain,
    get_session,
    input_messages_key="input",
    history_messages_key="history",
)

# Invocar com sessão
response = mentor.invoke(
    {"input": "Explica permeabilidade vs porosidade"},
    config={"configurable": {"session_id": "user-42"}},
)

O que aprendi

  • LCEL chains — Composição funcional com pipe (|) torna pipelines legíveis e testáveis.
  • Session memory — RunnableWithMessageHistory injeta contexto conversacional sem boilerplate.
  • Persona especializada — System prompt bem construído transforma LLM genérico em especialista de domínio.
  • Stateless deploy — Memory em dict permite escalar; migrar para Redis para produção.
